Vocabulary for the representation of the commercial expenditure data cube: Description back to index

For the generation of the commercial expenditure data cube, the W3C Data Cube vocabulary is reused. This vocabulary is a W3C recommendation for the publication of multidimensional data on the Web, which facilitates their structured representation, interoperability, and reuse. In addition, this vocabulary is compatible with the SDMX standard (Statistical Data and Metadata eXchange), which is widely used for the exchange of statistical data and metadata across organizations.
As shown in Figure 1, W3C Data Cube contains several concepts and properties that make it possible to describe statistical data at different levels of aggregation. In this vocabulary, the model is organized around the concept of qb:DataSet, which represents a dataset following a structure defined by a qb:DataStructureDefinition and materialized through a set of observations represented by qb:Observation. Specifically, this vocabulary defines a single dataset, represented by the instance edintgast:, and a single data structure, represented by the instance edintgast:dsd. This structure gathers all the dimensions and measures required to represent commercial expenditure information.
  • qb:DimensionProperty: component that identifies the dimensions of the observations, such as the reference administrative region, thematic sector, economic activity, reference period, or nationality. Dimensions usually take their values from code lists or classifications, typically represented using SKOS, or from other controlled resources.
  • qb:MeasureProperty: component that represents the measured value of the observed data. In this cube, the measure corresponds to the total monetary amount of commercial expenditure.
Several reference vocabularies are reused to define these dimensions and measures:
  1. SDMX vocabularies (Statistical Data and Metadata eXchange):
    • sdmx-dimension: reused as the basis for general statistical dimensions, especially through specialization relationships such as edintdimension:refAdminRegion and edintdimension:refPeriod.
    • sdmx-measure: reused as the basis for the cube measure, defined as a subproperty of sdmx-measure:obsValue.
  2. Territorial and temporal vocabularies:
    • esadm: reused to indicate that the dimension edintdimension:refAdminRegion may take values corresponding to municipalities, neighborhoods, or districts.
    • OWL-Time: reused to model the temporal dimension, establishing that edintdimension:refPeriod has time:Interval as its range.
  3. SKOS code lists:
    • The dimension edintdimension:sector reuses as a code list the public sector scheme published at datos.gob.es.
    • The dimension edintdimension:economicActivity reuses a controlled classification of economic activity, such as CNAE 2025.
    • The dimension edintdimension:nationality is used to represent the nationality associated with the commercial expenditure observation.
The commercial expenditure cube model is shown in Figure 2, where the dimensions and measures that compose it are presented.
The structure of the commercial expenditure cube is defined through the instance edintgast:dsd, which groups all the dimensions and measures of the model. In this case, the reference administrative region, sector, economic activity, reference period, and nationality have been defined as dimensions. As measure, the model includes the expenditure amount, which represents the total monetary amount of commercial expenditure made in a given context. 
edintgast: a qb:DataSet ; 
    qb:structure edintgast:dsd .

edintgast:dsd a qb:DataStructureDefinition ;
    qb:component
        [ qb:dimension edintdimension:refAdminRegion ; qb:order 1 ] ,
        [ qb:dimension edintdimension:sector ; qb:order 2 ] ,
        [ qb:dimension edintdimension:economicActivity ; qb:order 3 ] ,
        [ qb:dimension edintdimension:refPeriod ; qb:order 4 ] ,
        [ qb:dimension edintdimension:nationality ; qb:order 5 ] ,
        [ qb:measure edintmeasure:expenditureAmount ; qb:order 6 ] .

Example 1: Commercial expenditure amount by area, period, economic activity, sector, and nationality back to index

This example represents the data for the commercial expenditure amount in a given administrative region, for a specific reference period, broken down by economic activity, sector, and nationality. Since this vocabulary already defines the dataset edintgast: and the data structure edintgast:dsd, there is no need to generate new dataset or structure instances; the existing ones can be reused.
The steps required to represent a specific observation are described below. **1) Reuse of the dataset and the data structure:** This vocabulary contains a single dataset, represented by edintgast:, which is linked to the structure edintgast:dsd through the property qb:structure. 
edintgast: a qb:DataSet ; 
    qb:structure edintgast:dsd .
**2) Identification of the relevant dimensions and measure:** The structure edintgast:dsd contains the dimensions and measure of the commercial expenditure cube. In this example, the dimensions edintdimension:refAdminRegion, edintdimension:sector, edintdimension:economicActivity, edintdimension:refPeriod, and edintdimension:nationality are used together with the measure edintmeasure:expenditureAmount. It is important to note that the dimension edintdimension:refPeriod has time:Interval as its range, so the reference period must be represented through an explicit instance of a temporal interval. 
<http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/interval/2025-01>
    a time:Interval ;
    rdfs:label "January 2025"@en ,
               "Enero de 2025"@es .
**3) Definition of a dataset observation:** The final step consists of creating an instance of qb:Observation and linking it to the dataset through the property qb:dataSet. The corresponding values for the selected dimensions and measure are then assigned. 
<http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/observation/obs001>
    a qb:Observation ;
    qb:dataSet edintgast: ;
    edintdimension:refAdminRegion <http://vocab.linkeddata.es/datosabiertos/recurso/territorio/municipio/zaragoza> ;
    edintdimension:sector <https://datos.gob.es/kos/sector-publico/sector#comercio> ;
    edintdimension:economicActivity <http://lod.ine.es/kos/cnae/CNAE-2025/G> ;
    edintdimension:refPeriod <http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/interval/2025-01> ;
    edintdimension:nationality "Spanish"^^xsd:string ;
    edintmeasure:expenditureAmount "125430.75"^^xsd:decimal .
This observation indicates that, in the municipality of Zaragoza, during the temporal interval corresponding to January 2025, the commercial expenditure associated with the commerce sector, a specific economic activity, and Spanish nationality is 125,430.75. This example shows how a single cube observation combines several dimension values with a specific measure to represent commercial expenditure information. In the same way, the same structure can be reused to represent expenditure values for other administrative regions, reference periods, economic activities, sectors, or nationalities.
