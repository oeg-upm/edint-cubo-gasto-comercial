Vocabulario para la representación del cubo de datos de gasto comercial: Descripción volver a índice

Para la generación del cubo de datos de gasto comercial se reutiliza el vocabulario W3C Data Cube. Este vocabulario es una recomendación del W3C para la publicación de datos multidimensionales en la Web, lo que facilita su representación estructurada, interoperabilidad y reutilización. Además, este vocabulario es compatible con el estándar SDMX (Statistical Data and Metadata eXchange), ampliamente utilizado para el intercambio de datos y metadatos estadísticos entre organizaciones.
Como se muestra en la Figura 1, W3C Data Cube contiene varios conceptos y propiedades que permiten describir datos estadísticos en diferentes niveles de agregación. En este vocabulario, el modelo se organiza en torno al concepto de qb:DataSet, que representa un conjunto de datos que sigue una estructura definida mediante una qb:DataStructureDefinition y que se materializa a través de un conjunto de observaciones representadas mediante qb:Observation. En concreto, este vocabulario define un único conjunto de datos, representado por la instancia edintgast:, y una única estructura de datos, representada por la instancia edintgast:dsd. Esta estructura agrupa todas las dimensiones y medidas necesarias para representar información sobre gasto comercial.
  • qb:DimensionProperty: componente que identifica las dimensiones de las observaciones, como la región administrativa de referencia, el sector temático, la actividad económica, el periodo de referencia o la nacionalidad. Las dimensiones suelen tomar sus valores de listas de códigos o clasificaciones, normalmente representadas mediante SKOS, o de otros recursos controlados.
  • qb:MeasureProperty: componente que representa el valor medido de los datos observados. En este cubo, la medida corresponde al importe monetario total del gasto comercial.
Para definir estas dimensiones y medidas se reutilizan varios vocabularios de referencia:
  1. Vocabularios SDMX (Statistical Data and Metadata eXchange):
    • sdmx-dimension: se reutiliza como base para dimensiones estadísticas generales, especialmente mediante relaciones de especialización como edintdimension:refAdminRegion y edintdimension:refPeriod.
    • sdmx-measure: se reutiliza como base para la medida del cubo, definida como subpropiedad de sdmx-measure:obsValue.
  2. Vocabularios territoriales y temporales:
    • esadm: se reutiliza para indicar que la dimensión edintdimension:refAdminRegion puede tomar valores correspondientes a municipios, barrios o distritos.
    • OWL-Time: se reutiliza para modelar la dimensión temporal, estableciendo que edintdimension:refPeriod tiene como rango time:Interval.
  3. Listas de códigos SKOS:
    • La dimensión edintdimension:sector reutiliza como lista de códigos el esquema de sectores públicos publicado en datos.gob.es.
    • La dimensión edintdimension:economicActivity reutiliza una clasificación controlada de actividades económicas, como CNAE 2025.
El modelo del cubo de gasto comercial se muestra en la Figura 2, donde se presentan las dimensiones y medidas que lo componen.
La estructura del cubo de gasto comercial se define mediante la instancia edintgast:dsd, que agrupa todas las dimensiones y medidas del modelo. En este caso, la región administrativa de referencia, el sector, la actividad económica, el periodo de referencia y la nacionalidad se han definido como dimensiones. Como medida, el modelo incluye el importe del gasto, que representa la cantidad monetaria total del gasto comercial realizado en un contexto determinado. 
edintgast: a qb:DataSet ; 
    qb:structure edintgast:dsd .

edintgast:dsd a qb:DataStructureDefinition ;
    qb:component
        [ qb:dimension edintdimension:refAdminRegion ; qb:order 1 ] ,
        [ qb:dimension edintdimension:sector ; qb:order 2 ] ,
        [ qb:dimension edintdimension:economicActivity ; qb:order 3 ] ,
        [ qb:dimension edintdimension:refPeriod ; qb:order 4 ] ,
        [ qb:dimension edintdimension:nationality ; qb:order 5 ] ,
        [ qb:measure edintmeasure:expenditureAmount ; qb:order 6 ] .

Ejemplo 1: Importe del gasto comercial por área, periodo, actividad económica, sector y nacionalidad volver a índice

Este ejemplo representa el dato correspondiente al importe del gasto comercial en una región administrativa determinada, para un periodo de referencia específico, desglosado por actividad económica, sector y nacionalidad. Dado que este vocabulario ya define el conjunto de datos edintgast: y la estructura de datos edintgast:dsd, no es necesario generar nuevas instancias de conjunto de datos ni de estructura; se pueden reutilizar las ya existentes.
A continuación, se describen los pasos necesarios para representar una observación concreta. **1) Reutilización del conjunto de datos y de la estructura de datos:** Este vocabulario contiene un único conjunto de datos, representado por edintgast:, que está vinculado a la estructura edintgast:dsd mediante la propiedad qb:structure. 
edintgast: a qb:DataSet ; 
    qb:structure edintgast:dsd .
**2) Identificación de las dimensiones y la medida relevantes:** La estructura edintgast:dsd contiene las dimensiones y la medida del cubo de gasto comercial. En este ejemplo se utilizan las dimensiones edintdimension:refAdminRegion, edintdimension:sector, edintdimension:economicActivity, edintdimension:refPeriod y edintdimension:nationality, junto con la medida edintmeasure:expenditureAmount. Es importante señalar que la dimensión edintdimension:refPeriod tiene como rango time:Interval, por lo que el periodo de referencia debe representarse mediante una instancia explícita de intervalo temporal. 
<http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/interval/2025-01>
    a time:Interval ;
    rdfs:label "January 2025"@en ,
               "Enero de 2025"@es .
**3) Definición de una observación del conjunto de datos:** El último paso consiste en crear una instancia de qb:Observation y vincularla al conjunto de datos mediante la propiedad qb:dataSet. A continuación, se asignan los valores correspondientes a las dimensiones seleccionadas y a la medida. 
<http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/observation/obs001>
    a qb:Observation ;
    qb:dataSet edintgast: ;
    edintdimension:refAdminRegion <http://vocab.linkeddata.es/datosabiertos/recurso/territorio/municipio/zaragoza> ;
    edintdimension:sector <https://datos.gob.es/kos/sector-publico/sector#comercio> ;
    edintdimension:economicActivity <http://lod.ine.es/kos/cnae/CNAE-2025/G> ;
    edintdimension:refPeriod <http://vocab.linkeddata.es/datosabiertos/recurso/comercio/cubo-gasto-comercial/interval/2025-01> ;
    edintdimension:nationality "Spanish"^^xsd:string ;
    edintmeasure:expenditureAmount "125430.75"^^xsd:decimal .
Esta observación indica que, en el municipio de Zaragoza, durante el intervalo temporal correspondiente a enero de 2025, el gasto comercial asociado al sector comercio, a una actividad económica concreta y a la nacionalidad española es de 125.430,75. Este ejemplo muestra cómo una única observación del cubo combina varios valores de dimensiones con una medida específica para representar información sobre gasto comercial. Del mismo modo, la misma estructura puede reutilizarse para representar valores de gasto en otras regiones administrativas, periodos de referencia, actividades económicas, sectores o nacionalidades.
